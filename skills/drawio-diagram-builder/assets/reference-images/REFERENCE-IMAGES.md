# Bundled Reference Images

These images are bundled as style and layout references for computer-science paper figures. They are not final output assets.

Use them only to infer visual language when the user asks for a top-conference-style research figure but provides weak or missing style references. Do not embed these raster images into the final `.drawio` unless the user explicitly asks for a raster reference layer.

## Inventory

| file | style use |
|------|-----------|
| `topconf-memory-routing.png` | Dense ML/system method figure with hierarchy, equations, modality icons, muted containers, and explicit routing semantics. |
| `topconf-knowledge-mining-pipeline.png` | Multi-panel algorithm pipeline with matrices, prompt blocks, iteration axes, dashed groups, and compact academic labels. |
| `topconf-ccf-adapter-architecture.png` | Multimodal architecture figure with subfigures, token sequences, trainable/frozen legend, experts, adapters, and losses. |
| `topconf-handdrawn-rl-pipeline.png` | Stage-based training/RL workflow with dashed containers, product/content inputs, reward models, serving block, and sketch-like emphasis. |

## Usage Rules

- Treat the images as visual references, not content sources.
- Preserve the user's scientific terms, modules, and flow over the reference image's topic.
- Recreate reusable style traits with editable draw.io primitives: containers, token bars, legends, matrices, arrows, highlights, and icon approximations.
- If content is underspecified, use these images only to choose layout and style; ask for missing scientific components instead of inventing claims.
